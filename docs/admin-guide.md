## 1. Доступ к панели администратора

Войдите в продукт под учётной записью с правами администратора. В левом боковом меню появятся дополнительные пункты:

- **Дашборд** — аналитика по пользователям
- **Журнал обработок** — история транскрибаций
- **Настройки ИИ** — выбор AI-моделей
- **Пользователи** — управление аккаунтами
- **Подписки** — тарифы и оплата
- **Архив** — данные удалённых пользователей

> Если меню не появилось — обратитесь к техническому администратору для выдачи прав `isAdmin`.

## 2. Дашборд

**Адрес:** `/admin/dashboard`

Главный экран администратора. Обновляется автоматически каждую минуту.

### Карточки итогов (верхняя строка)

| Карточка | Что показывает |
|---|---|
| Пользователей | Общее количество аккаунтов (без архивариуса) |
| Встреч обработано | Число успешно завершённых транскрибаций за всё время |
| Стоимость AI | Суммарная себестоимость API-запросов в долларах США |
| Оплачено подписок | Суммарная сумма успешных платежей в рублях |

### Таблица пользователей

Каждая строка — один пользователь. Кликните на заголовок колонки для сортировки.

| Колонка | Описание |
|---|---|
| Пользователь | Email, имя, бейджи ADMIN и БЛОК |
| Зарегистрирован | Дата создания аккаунта |
| Дней | Количество дней с момента регистрации |
| Сессий | Сколько раз пользователь входил в продукт |
| Последняя активность | Дата и время последнего действия в продукте |
| Подписка | Текущий тариф и дата окончания |
| Встреч | Количество обработанных встреч |
| Часов аудио | Суммарная длительность загруженных файлов |
| AI, $ | Себестоимость AI-операций (для внутренней аналитики) |
| Оплачено, ₽ | Сумма всех успешных платежей пользователя |

## 3. Пользователи

**Адрес:** `/admin/users`

### Просмотр списка

Используйте строку поиска для фильтрации по email или имени. В каждой карточке пользователя видно:

- Email, имя
- Бейджи: **ADMIN** (администратор), **ЗАБЛОКИРОВАН** (не может войти)
- Дата регистрации
- Число сессий и время последней активности
- Количество встреч

### Создание пользователя

Нажмите кнопку **«+ Добавить»** в правом верхнем углу раздела.

Заполните поля:

| Поле | Описание |
|---|---|
| Email | Рабочий адрес электронной почты |
| Пароль | Минимум 6 символов |
| Имя | Необязательно |
| Admin | Чекбокс — выдаёт права администратора |

> Пользователи, созданные через admin-панель, сразу могут входить в продукт — подтверждение email не требуется.

### Редактирование пользователя

Нажмите **«⋯»** → **«Редактировать»**. Доступно изменение:

- Email
- Имя
- Права администратора (Admin)
- Статус блокировки (Заблокирован)
- Пароль (поле новый пароль — необязательное)

Нажмите **«Сохранить»** для применения изменений.

### Блокировка и разблокировка

**Быстрый способ:** нажмите **«⋯»** → **«Заблокировать»** / **«Разблокировать»**.

Заблокированный пользователь:
- Видит сообщение «Ваш аккаунт заблокирован. Обратитесь к администратору»
- Если уже залогинен — будет автоматически выброшен при следующем обращении

**Через форму редактирования:** откройте редактирование → отметьте чекбокс «Заблокирован» → «Сохранить».

### Удаление пользователя

Нажмите **«⋯»** → **«Удалить»** → подтвердите.

**Что происходит с данными:**

- Все встречи пользователя сохраняются и переходят к архивариусу
- Workspace пользователя помечается как архивный (видно в разделе «Архив»)
- Личные данные (пароль, токены) удаляются безвозвратно

> **Важно:** перед первым удалением пользователя необходимо создать архивариуса в разделе «Архив».

## 4. Подписки

**Адрес:** `/admin/subscriptions`

### Назначение подписки

1. Выберите пользователя из выпадающего списка
2. Нажмите **«+ Назначить подписку»**
3. Заполните форму:

| Поле | Описание |
|---|---|
| Тариф | Бесплатный / Стартер 490 ₽ / Про 990 ₽ |
| Статус | Активна / Пробный / Бесплатный |
| Период | Месяц / Год (не обязателен для бесплатного) |
| Действует до | Дата окончания подписки |
| Комментарий | Внутренняя заметка (не видна пользователю) |

**Опциональная запись платежа:**

| Поле | Описание |
|---|---|
| Сумма, ₽ | Сумма полученного платежа |
| Комментарий | Способ оплаты или примечание |

4. Нажмите **«Назначить подписку»**

### История подписок пользователя

После выбора пользователя отображается список всех его подписок с датами и суммами платежей.

### Отмена подписки

В истории подписок нажмите **«Отменить»** напротив активной подписки. Доступно для статусов: Активна, Пробный, Льготный период.

### Статусы подписок

| Статус | Описание |
|---|---|
| 🟢 Бесплатный | Бесплатный тариф без ограничения по времени |
| 🔵 Пробный | Временный бесплатный доступ |
| 🟢 Активна | Оплаченная подписка действует |
| 🟡 Льготный период | Подписка истекла, но данные ещё доступны (14 дней) |
| 🔴 Истекла | Подписка и льготный период закончились |
| 🔴 Отменена | Подписка отменена вручную |

## 5. Архив

**Адрес:** `/admin/archive`

### Создание архивариуса

При первом открытии раздела нажмите **«Создать архивариуса»**. Это нужно сделать **один раз** перед первым удалением пользователя.

Архивариус — системный аккаунт `archivist@system.internal`, который:
- Принимает встречи и данные удалённых пользователей
- Не может войти в продукт (заблокирован)
- Не отображается в основном списке пользователей

### Workspace-сироты

После создания архивариуса список показывает все workspace удалённых пользователей.

Для каждого workspace видно:
- Название workspace
- Email бывшего владельца (кто был удалён)
- Количество встреч

### Переназначение встреч

Чтобы передать встречи удалённого пользователя активному сотруднику:

1. Найдите нужный workspace в списке
2. Выберите пользователя из выпадающего списка
3. Нажмите **«Передать»**

**Результат:** все встречи из архивного workspace появятся в списке встреч выбранного пользователя. Workspace исчезнет из раздела «Архив».

## 6. Журнал обработок

**Адрес:** `/admin/runs`

Показывает историю всех транскрибаций всех пользователей.

### Фильтры

| Фильтр | Назначение |
|---|---|
| Статус | pending / running / completed / failed / partial_failed |
| User ID | Фильтр по идентификатору пользователя |
| User email | Поиск по email (частичное совпадение) |
| От / До | Фильтр по дате запуска |
| Ошибки | Только с ошибками / Только без ошибок |

Список обновляется автоматически каждые 5 секунд.

### Карточка запуска

Нажмите **«Детали»** для просмотра подробностей:

- Статус и продолжительность обработки
- Какие AI-модели использовались
- Стоимость каждого шага
- Детальный лог шагов с кодами ошибок
- Кнопка **«Перезапустить»** — создаёт новый запуск для той же встречи

**Статусы запуска:**

| Статус | Описание |
|---|---|
| pending | В очереди, ожидает worker |
| running | Обрабатывается прямо сейчас |
| completed | Завершён успешно |
| failed | Завершён с критической ошибкой |
| partial_failed | Часть шагов выполнена, часть — с ошибкой |

> Если встреча долго в статусе `pending` — worker не запущен. Обратитесь к техническому администратору.

## 7. Настройки AI-моделей

**Адрес:** `/admin/models`

Позволяет переключать AI-провайдеры без изменения кода и перезапуска сервера.

### Транскрипция

Выберите и активируйте один из провайдеров:

| Провайдер | Стоимость | Диаризация | Необходимый ключ |
|---|---|---|---|
| Groq LLaMA | Бесплатно (лимит 7200 с/час) | Нет | `GROQ_API_KEY` |
| Mock (разработка) | Бесплатно | Нет | — |
| OpenAI Whisper | $0.006/мин | Нет | `OPENAI_API_KEY` |
| Deepgram nova-3 | $0.0059/мин | Да | `DEEPGRAM_API_KEY` |
| AssemblyAI | $0.0062/мин | Да | `ASSEMBLYAI_API_KEY` |
| Gladia | Бесплатно 10 ч/мес | Да | `GLADIA_API_KEY` |
| Google Gemini Flash | $0.10/1M токенов | Да | `GEMINI_API_KEY` |

> **Диаризация** — автоматическое определение кто говорит (разделение по спикерам). Рекомендуется для записей с несколькими участниками.

### Постобработка (саммари, задачи, Q&A)

| Провайдер | Стоимость | Необходимый ключ |
|---|---|---|
| Groq LLaMA-3.3-70b | Бесплатно | `GROQ_API_KEY` |

| OpenAI GPT-4o | $2.50/1M токенов | `OPENAI_API_KEY` |
| OpenAI GPT-4o-mini | $0.15/1M токенов | `OPENAI_API_KEY` |
| Google Gemini Flash | $0.10/1M токенов | `GEMINI_API_KEY` |

**Рекомендуемая конфигурация для старта:**
- Транскрипция: **Groq** (бесплатно, быстро)
- Постобработка: **Groq** (бесплатно, хорошее качество на русском)

## 8. Типовые сценарии

### Новый пользователь не может войти

**Причина:** не подтверждён email.

**Решение:** в разделе «Пользователи» найдите пользователя. Если видите, что проблема в верификации — обратитесь к техническому администратору для ручного исправления в БД, либо попросите пользователя повторно зарегистрироваться.

### Сотрудник уволился, нужно передать его встречи

1. Перейдите в «Пользователи» → найдите сотрудника → **«⋯»** → **«Удалить»**
2. Перейдите в **«Архив»** → найдите workspace удалённого сотрудника
3. Выберите нового владельца → нажмите **«Передать»**
4. Новый сотрудник увидит все встречи в своём списке

### Назначить пробный период пользователю

1. Перейдите в «Подписки»
2. Выберите пользователя
3. Нажмите «Назначить подписку»
4. Выберите нужный тариф, статус **«Пробный»**
5. Укажите дату окончания (например, через 14 дней)
6. Нажмите «Назначить подписку» (без записи платежа)

### Зафиксировать оплату подписки

1. Перейдите в «Подписки» → выберите пользователя
2. Нажмите «Назначить подписку»
3. Выберите тариф, статус **«Активна»**, период (Месяц/Год), дату окончания
4. В блоке «Платёж» укажите сумму и примечание (например, «Оплата Сбербанк»)
5. Нажмите «Назначить подписку»

### Транскрибация зависла

1. Перейдите в «Журнал обработок»
2. Найдите встречу (фильтр по email пользователя)
3. Если статус `pending` долгое время — worker не запущен, обратитесь к техническому администратору
4. Если статус `failed` — откройте детали → посмотрите код ошибки → нажмите **«Перезапустить»**

### Переключить AI-модель (например, на более дешёвую)

1. Перейдите в «Настройки ИИ»
2. Выберите нужную карточку транскрипции или постобработки
3. Нажмите **«Активировать»**

Все новые транскрибации будут использовать новую модель. Уже созданные не затрагиваются.

## 9. Справочник тарифов

| Тариф | Цена | Минут аудио/мес | Встреч/мес | Хранение данных | Льготный период |
|---|---|---|---|---|---|
| Бесплатный | 0 ₽ | 60 мин | 5 | 90 дней | нет |
| Стартер | 490 ₽/мес · 4 900 ₽/год | 300 мин (5 ч) | 20 | 180 дней | 14 дней |
| Про | 990 ₽/мес · 9 900 ₽/год | Без лимита | Без лимита | 365 дней | 14 дней |

**Льготный период** — время после окончания подписки, в течение которого данные сохраняются и доступны только для чтения.
