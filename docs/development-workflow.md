# Как развивать и поддерживать приложение

## Твоя идея vs. как это работает на практике

Ты предложил:
> 1. Вношу изменения в код
> 2. Прошу обновить GitHub
> 3. Прошу произвести изменения на сервере

**Это верно, но шаги 1 и 2 — одна операция.** Когда я правлю код, я сразу же делаю коммит и пуш в GitHub. Разделять это не нужно. Итоговая схема:

```
Ты объясняешь задачу
      ↓
Я правлю код + делаю коммит + пушу в GitHub
      ↓
Ты говоришь "задеплой на сервер"  ← единственный отдельный шаг
      ↓
Я выполняю деплой одной командой
```

---

## Три сценария работы

### Сценарий A — Новая функция или исправление

**Ты:** Опиши, что хочешь изменить. Например:
- "Добавь поле 'участники встречи' в карточку"
- "Исправь: при экспорте в Telegram обрезает текст"
- "Сделай тёмную тему"

**Я:**
1. Изучаю код
2. Вношу изменения
3. Делаю коммит и пушу в GitHub автоматически

**Ты:** "Задеплой на сервер"

**Я:** Подключаюсь к серверу и выполняю одну команду:
```bash
deploy-meeting-ai
```

Готово. Занимает ~30 секунд.

---

### Сценарий B — Изменение настроек (API-ключи, лимиты)

Если нужно поменять переменную окружения (например, добавить новый API-ключ):

**Ты:** "Добавь новый ключ для сервиса X"

**Я:** Подключаюсь к серверу, правлю `.env`, перезапускаю:
```bash
# Редактирую /opt/meeting-ai/apps/backend/.env
pm2 restart meeting-ai-api meeting-ai-worker
```

Это **не требует** пуша в GitHub — `.env` не хранится в репозитории по соображениям безопасности.

---

### Сценарий C — Изменение структуры базы данных

Если добавляется новая таблица или поле (миграция Prisma):

**Я:** Создаю миграцию локально, пушу в GitHub.

При деплое команда `deploy-meeting-ai` автоматически применит миграцию на сервере (`prisma migrate deploy`).

---

## Что делает команда `deploy-meeting-ai` на сервере

```
[1/5] git pull origin main          — скачивает изменения из GitHub
[2/5] tsc (shared)                  — пересобирает общий пакет типов
[3/5] prisma migrate deploy         — применяет новые миграции БД
[4/5] pnpm build (frontend)         — пересобирает фронтенд
[5/5] pm2 restart all               — перезапускает API и воркер
```

Запустить вручную (подключившись по SSH):
```bash
deploy-meeting-ai
```

---

## Что нужно от тебя — коротко

| Ситуация | Что сказать мне |
|----------|-----------------|
| Хочешь новую функцию | Опиши словами, что должно работать |
| Нашёл баг | Опиши, что происходит не так и при каких условиях |
| Хочешь поменять дизайн | Опиши или покажи скриншот |
| Готов обновить сервер | "Задеплой на сервер" |
| Проблема на сервере | "Проверь логи / сервер лежит" |

---

## Что НЕ нужно делать вручную

- ❌ Открывать GitHub и делать коммиты самому — я делаю это автоматически
- ❌ Вручную копировать файлы на сервер — для этого есть `deploy-meeting-ai`
- ❌ Останавливать/запускать PM2 при деплое — скрипт делает `restart`, не `stop`
- ❌ Трогать БД напрямую без необходимости — для обычных задач это не нужно

---

## Как я подхожу к задачам (для понимания)

1. **Сначала читаю код** — не пишу "вслепую", всегда смотрю на существующую структуру
2. **Минимальные изменения** — не переписываю то, что работает
3. **Коммит сразу** — изменения в GitHub появляются в ту же сессию
4. **Деплой отдельно** — только по твоей команде, чтобы ты контролировал момент

---

## Полезные команды (для самостоятельной диагностики)

Подключись по SSH и выполни:

```bash
# Посмотреть статус всех процессов
pm2 list

# Логи в реальном времени
pm2 logs

# Только ошибки бэкенда
pm2 logs meeting-ai-api --err --lines 50

# Перезапустить всё вручную
pm2 restart all

# Обновить сервер вручную (без моей помощи)
deploy-meeting-ai
```

```
                    РАЗРАБОТКА

│                                                 │
│   Ты: "Хочу X"                                  │
│         ↓                                       │
│   Я: правлю код → коммит → push в GitHub        │
│         ↓                                       │
│   Ты: "Задеплой"                                │
│         ↓                                       │
│   Я: ssh → deploy-meeting-ai (30 сек)           │
│         ↓                                       │
│   Сайт обновлён на http://150.241.107.20        │
│                                                 │
└─────────────────────────────────────────────────┘

                   ДИАГНОСТИКА
┌─────────────────────────────────────────────────┐
│                                                 │
│   Ты: "Что-то сломалось / не работает X"        │
│         ↓                                       │
│   Я: подключаюсь, читаю логи, нахожу причину    │
│         ↓                                       │
│   Исправляю код или конфиг → деплой             │
│                                                 │
└─────────────────────────────────────────────────┘
```
